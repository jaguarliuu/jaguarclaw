JaguarClaw Visual Architecture Kit

更适合演示、评审与对外展示的 HTML 架构图页

这份页面把 JaguarClaw 的核心流程图、时序图和对外展示图放进了一个更完整的视觉包装里。 你可以直接浏览器打开，用于技术评审、项目汇报、产品说明或客户沟通。

适用场景

研发架构评审：解释 Gateway、Runtime、LLM、Tools、Memory 的关系
产品汇报：展示多 Agent、工具、技能、自动化与部署能力
客户演示：用更友好的产品图和部署拓扑说明系统边界

使用方式

直接打开当前 HTML 文件即可预览
依赖本地 jaguarclaw-ui/node_modules/mermaid 渲染
适合继续导出截图或放进宣传材料

agent.run 主链路时序图

展示一次会话从前端发起、进入 Gateway、投递到 Agent Runtime，再到模型推理、工具执行、事件回推和状态落库的完整过程。

Execution Flow
sequenceDiagram autonumber participant U as User participant FE as UI / Desktop participant GW as Gateway RPC participant AR as Agent Runtime participant CB as Context Builder participant LLM as LLM Client participant TR as Tool Registry / Dispatcher participant DB as State Store U->>FE: 输入任务并发送 FE->>GW: agent.run(sessionId, prompt, modelSelection) GW->>DB: 创建 run + 更新状态 queued GW->>AR: 投递到 session lane AR->>DB: 状态 running AR->>CB: 组装 system prompt + history + memory + skills CB-->>AR: messages AR->>LLM: stream(messages, tools, tool_choice) LLM-->>AR: assistant.delta AR-->>GW: assistant.delta 事件流 GW-->>FE: 推送流式内容 alt 模型发起工具调用 LLM-->>AR: tool_calls AR->>TR: 执行工具 TR->>DB: 记录 tool.call / tool.result TR-->>AR: observation AR->>LLM: 下一轮推理 end LLM-->>AR: final answer / finish_reason AR->>DB: 保存 assistant message + run done AR-->>GW: lifecycle.end GW-->>FE: 最终完成事件
适合谁看后端、前端、测试、实施。
强调什么事件流、状态切换、模型与工具的协同过程。
推荐用途问题排查、联调说明、流程培训。

工具调用闭环时序图

突出 JaguarClaw 的 agentic 核心：模型如何发起工具调用，工具如何返回 observation，以及人在需要时怎样介入确认。

Agentic Core
sequenceDiagram autonumber participant RT as ReAct Loop participant LLM as OpenAI-Compatible Model participant TD as Tool Dispatcher participant Tool as Built-in Tool / MCP Tool participant HITL as Confirm Layer RT->>LLM: 提交当前上下文 + 工具定义 LLM-->>RT: 返回 tool_calls RT->>TD: dispatch(toolName, arguments) alt 需要人工确认 TD->>HITL: 发起 confirm_request HITL-->>TD: approve / reject end TD->>Tool: execute(arguments) Tool-->>TD: result / error TD-->>RT: observation RT->>LLM: 回灌 tool result，继续推理 LLM-->>RT: 最终回答
适合谁看架构评审、AI 平台、售前方案。
强调什么ReAct 闭环、HITL、安全控制与扩展能力。
推荐用途解释为什么系统不只是“聊天 UI + LLM API”。
